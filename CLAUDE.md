# compressiform

2-day game jam project. Theme: **signal**. Side-view 2D puzzle: the player is given a message and a stone tablet, and must compress the message as much as possible with minimal loss of information. Fork of an Odin + Raylib top-down shooter starter (Karl Zylinski's template); shooter-specific content has been stripped.

## Build & run

**Primary submission target ‚Äî web:**
```
make web           # build wasm bundle into build/web/
make web-deploy    # build web AND deploy to GitHub Pages (gh-pages branch)
```
No hot reload in web; rebuild each iteration.

**Default dev loop ‚Äî native with hot reload:**
```
make hot-reload-run
```
Edits to game code swap the `game.dylib` without restarting the runner. In-game keys: `F5` force-reload, `F4` reload shader, `` ` `` frame-step while paused.

**Other targets:**
- `make run` ‚Äî plain native debug
- `make speed` ‚Äî optimized native
- `make release` ‚Äî shippable native build (also runs `atlas`)
- `make debug` / `make mem` / `make perf` ‚Äî debugger / leak-tracking / perf-timing variants
- `make atlas` ‚Äî regenerate texture atlas (required after adding/editing PNGs in `source/textures/`)
- `make palette` ‚Äî open the palette viewer (zoom/pan, hover to inspect, click to copy name)
- `make help` ‚Äî full list

**Web gotchas:** hot reload is native-only; sounds are `#load_directory`-baked so new audio requires a full rebuild; asset paths are virtualized through Emscripten.

## Where the game lives

All puzzle-game code goes in `source/compressiform.odin`:

- `game_start()` ‚Äî initialization. Empty stub. Spawn initial objects, load the first puzzle here.
- `game_update(dt)` ‚Äî per-frame logic. Empty stub. Input, puzzle state, scoring.
- `GameSpecificGlobalState` struct ‚Äî add puzzle state fields here (current message, tablet contents, score, etc.).

Engine main loop is `game_step()` in `source/main.odin`: physics ‚Üí collision ‚Üí `game_update` ‚Üí animation ‚Üí render.

## Engine concepts (only what's relevant for this game)

- **GameObject** (`source/object_types.odin`): handle-based entity in the `GameObjects` HandleMap. Has transform, render info, optional hitbox, animation state, and a `variant` union where you attach puzzle-specific data. Spawn via `spawn_object()`.
- **Tags** (bit_set): flip `.Sprite`, `.Text`, `.Collide` etc. to opt an object into subsystems.
- **Render layers**: 128 indexed layers control draw order.
- **Camera**: `game.main_camera` (position + zoom). Objects parented to `screen_space_parent_handle` render in UI space (don't move with the camera).
- **Text rendering**: set `.Text` tag + `TextRenderInfo` (font, size, color, alignment) on a GameObject. Draw code in `source/render.odin` (~L214). This is the path for the on-tablet message, the source message, and all UI.
- **Input**: direct Raylib calls (`rl.IsKeyPressed(.SPACE)`, `rl.GetCharPressed()`, etc.) inside `game_update`. No abstraction layer ‚Äî keep it simple.

## Assets

- **Fonts** (`source/fonts/`, selected by `get_font(FontName)` enum ‚Äî regenerated by `make atlas`):
  - `Rye` ‚Äî display/titles
  - `GermaniaOne` ‚Äî UI headings, buttons
  - `MedievalSharp` ‚Äî body/dialogue
  - `AtkinsonHyperlegible-Bold` ‚Äî set as `MAIN_FONT` in `source/main.odin` (default/fallback)
  - `cmunit` ‚Äî monospace
- **Palette** (`source/palettes/caveman.gpl`, documented in `source/palettes/COLORS.md`): 12 colors ‚Äî earth tones dominant (Charcoal, Mud Brown, Clay, Ochre, Sandstone, Bone, Ash‚Ä¶), accents in Ember Red / Fire Orange / Moss / Night Sky. Use `make palette` to browse.
- **Textures** (`source/textures/`): basic tiles plus 5 procedurally generated stone tablets at `tablets/tablet{50,75,100,125,150}.png`. **After adding or changing a PNG here, run `make atlas`** ‚Äî `source/atlas.odin` is generated, don't edit by hand.
- **Tablet generator**: `scripts/make_tablet.py` (Pillow + scipy) ‚Äî regenerate tablets at different sizes/tints by editing parameters.
- **Sounds**: drop into a `sounds/` directory; compiled in via `#load_directory`.

## Jam-mode working rules

- **Scope discipline**: 2 days total. MVP > polish > features. If a mechanic takes more than an hour to prototype, cut it down further.
- **Ship the web build**: test `make web` early and often; `make web-deploy` pushes to GitHub Pages.
- **Theme**: *signal* ‚Äî interpreted here as the message being transmitted. The puzzle is the receiver's decision about what signal to preserve when bandwidth (the tablet) is finite.
